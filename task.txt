Требуется реализовать компонент на VueJS/ReactJs/Angular, выводящий
информацию о результатах поиска
объектов недвижимости по части кадастрового номера из открытых данных
Росреестра.

Выглядит компонент так:

Имеется строка ввода с лейблом "Кадастровый номер" и кнопка "Найти".
Строка ввода допускает вводить только цифры либо двоеточие, минимальная
длина 5 символов (пример: 50:01:, 54:01:23001, 77:12).

При нажатии на кнопку после ввода части кад. номера происходит GET
запрос на URL
https://rosreestr.ru/fir_lite_rest/api/gkn/fir_objects/<тут введённое
значение кад. номера>* (обратите внимание, что к значению из поля
приписывается звёздочка). В ответе будет список JSON объектов.
Этот список отображается в табличке со следующими колонками:

1. "#" - просто номер строки по порядку начиная с 1ки.

2. "Кад. номер" - значение поля "objectCn" из JSON объекта элемента списка

3. "Тип" - значение поля "objectType", преобразованное следующим образом:

     parcel -> выводим "Земельный участок",

     flat -> выводим "Квартира",

     building -> выводим "Здание",

     construction -> выводим "Сооружение",

     любое другое значение -> оставляем как есть

4. "Адрес" - значение поля "addressNotes"


Кроме того, кад. номер во второй колонке должен быть кликабельной
ссылкой, по нажатию на которую происходит GET запрос на URL с данными по
конкретному объекту и показ модального окна с этими данными.

URL следующий (возвращает JSON объект с описанием объекта недвижимости):

https://rosreestr.ru/fir_lite_rest/api/gkn/fir_lite_object/<тут значение
поля objectId>

Модальное окно содержит табличку со следующей информацией:

- "Дата записи" : Значение поля "objectData"."dateCreated"

- "Наименование" : Значение из "objectData"."name", если поля нет, то
значение из "objectData"."objectDesc"

- "Обременения": true = "Зарегистрированы", false = "Не
зарегистрированы". В зависимости от поля "type" значение из:

     parcel -> "parcelData"."encumbrancesExists",

     building -> "building"."encumbrancesExists",

     construction -> "construction"."encumbrancesExists",

     flat -> "flat"."encumbrancesExists",

     для прочих типов ничего не выводим


Требований по дизайну внешнего вида компонента нет.
